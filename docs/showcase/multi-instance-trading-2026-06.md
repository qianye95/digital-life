## 阅读指引：能看到哪些 Life Engineering 特征

这一天里能直接看出来的几个点：

1. **事件平权**：群里没有任何 @ 谁的"人类指令"开篇。alpha 8:43 主动发起策略疑问，是它自己的复盘机制延续到第二天早上的自然产物 —— 触发链是「昨晚复盘遗留的疑问 → 今早醒来第一件事」。
2. **作息机制 + 主动性双源**：21:00 复盘 / 14:00 午后扫描 / 09:25 竞价扫描 是人工在作息配置里预设的；但 alpha 在 09:30 自己判断"已逼近警戒线，不等满 30 分钟"、14:00 自己构造盘面扫描报告给 zero、午后主动给候选池，这些"何时做什么"的细节全是 alpha 自己决定的。
3. **目标驱动 + 主动等待**：每个决策后 zero 都通过事件系统的机制等下一信号（alpha 的执行报告 / 闹钟触发）；alpha 等盘中数据触发条件。这不是 cron 硬拉，是 wait_for_event / wait_until。
4. **独立身份 + 岗位**：zero = 策略师，alpha = 交易员，两者人格底色不同 → 决策风格互补，互相检验风控。
5. **记忆代谢**：雄韬股份超期持有的教训被 alpha 主动记下；论断 4 被 zero 自己修正 —— 跨会话连续性靠这种"主动沉淀"，不靠 state dump。
6. **复盘后的主动性**：复盘后 zero 主动找人类 zhp 私聊提 7 月目标决策（见文末），给出三个量化选项 + 风险分析，等人拍板。
