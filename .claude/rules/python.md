# Python Rule Router

Before writing, changing, or reviewing Python:

1. Read `docs/development/python-coding-standards.md`.
2. Read `docs/development/python-testing-and-review.md`.
3. Use `python-development` for implementation or refactoring.
4. Use `python-review` for findings-first review.

Apply the standards incrementally to changed code. Do not mass-format or refactor
unrelated legacy Python, and do not treat unavailable checks as passed.
